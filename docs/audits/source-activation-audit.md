# Runtime Source Activation Audit

Date: 2026-04-23  
Branch: `docs/source-activation-audit`  
Scope: static code audit only; no runtime feed fetches, no preview checks, and no source activation changes.

## Executive Summary

Verdict counts:

| Verdict | Count |

| Confirmed Live | 6 |
| Likely Live | 3 |
| Dormant but Partially Wired | 12 |
| Dormant | 10 |
| Insufficient Evidence | 0 |

Top findings:

1. `src/lib/source-catalog.ts` is not the runtime activation source of truth. The catalog defines 25 recommended-source entries, but default runtime ingestion is assembled through `MVP_DEFAULT_PUBLIC_SOURCE_IDS`, donor defaults, and probationary runtime feed IDs in separate files (`src/lib/source-catalog.ts:19`, `src/lib/demo-data.ts:117-123`, `src/adapters/donors/registry.ts:765-774`).
2. The prior "three confirmed-live" belief is only partly supported. TechCrunch is live, but through runtime ID `source-techcrunch`, not catalog ID `techcrunch` (`src/lib/source-catalog.ts:184-198`, `src/lib/demo-data.ts:58-64`, `src/lib/demo-data.ts:117-123`). GDELT Finance Monitor is not confirmed live: it is cataloged and has GDELT fallback logic, but no default runtime source ID was found (`src/lib/source-catalog.ts:370-386`, `src/lib/data.ts:2199-2215`, `src/lib/data.test.ts:171-210`). NewsAPI Business is conditional runtime metadata, but it is not in the source catalog and is not part of the MVP default source set (`src/lib/demo-data.ts:102-114`, `src/lib/demo-data.ts:117-123`).
3. `src/lib/ranking.ts` does not contain per-source authority entries in the current code. Source authority/preference signals live in `src/lib/source-policy.ts` and are consumed by event intelligence (`src/lib/ranking.ts:27-53`, `src/lib/source-policy.ts:10-93`, `src/lib/event-intelligence.ts:785-810`). In this audit, "ranking authority" means a matching `SOURCE_PREFERENCE_RULES` entry or donor `trustTier` metadata.

## Methodology

Files reviewed:

- Catalog inventory: `src/lib/source-catalog.ts`
- Runtime public/default source assembly: `src/lib/demo-data.ts`, `src/lib/data.ts`, `src/lib/pipeline/index.ts`, `src/lib/pipeline/ingestion/index.ts`
- Donor runtime registry: `src/adapters/donors/registry.ts`, `src/adapters/donors/openclaw/index.ts`, `src/adapters/donors/horizon/index.ts`, `src/adapters/donors/after_market_agent/index.ts`, `src/adapters/donors/fns/index.ts`
- Feed protocol handlers: `src/lib/rss.ts`
- Env gates: `src/lib/env.ts`
- Ranking/source authority: `src/lib/ranking.ts`, `src/lib/event-intelligence.ts`, `src/lib/source-policy.ts`
- Guardrail tests and prior source-governance assertions: `src/lib/source-defaults.test.ts`, `src/lib/source-catalog.test.ts`, `src/lib/source-policy.test.ts`, `src/lib/data.test.ts`

Search patterns used:

- Exact source IDs from `recommendedSources`, excluding `src/lib/source-catalog.ts`
- Display names and feed URLs for logical matches where runtime IDs differ from catalog IDs
- Protocol handlers: `thenewsapi://`, `newsapi://`, ordinary RSS/HTTP, and GDELT URL/name detection
- Runtime assembly references: `demoSources`, `MVP_DEFAULT_PUBLIC_SOURCE_IDS`, `DEFAULT_DONOR_FEED_IDS`, `PROBATIONARY_RUNTIME_FEED_IDS`, `getMvpDefaultPublicSources`, `getDefaultDonorFeeds`, `getProbationaryRuntimeFeeds`
- Authority references: `SOURCE_PREFERENCE_RULES`, `classifySourcePreference`, `trustTier`

Static-analysis limitations:

- This audit did not fetch live feeds, execute the pipeline, or inspect production/preview deployments.
- Exact source-ID absence is based on static search. It is strong evidence that a source is not explicitly wired by that ID, but it cannot prove that a dynamically supplied account source will never use the same feed.
- Several sources are logically live under different runtime IDs. Those rows call out the ID mismatch instead of treating catalog metadata as runtime truth.
- `demoSources` includes non-default sources. This audit distinguishes "present in runtime metadata" from "part of the default public ingestion set" because `getMvpDefaultPublicSources()` only returns the explicit MVP default IDs (`src/lib/demo-data.ts:117-136`).
