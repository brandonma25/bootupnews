# ADR 003: Decoupled Article-Body Extraction with Same-Cycle Restage

The decision is to unblock the `source_accessibility` gate — which starves the editorial queue of genuine breaking news that arrives as short RSS abstracts — by adding two decoupled cron legs rather than changing the gate, the thresholds, or the main ingestion cron. CRON-1 (`/api/cron/extract-article-bodies`) fetches and caches full article bodies for the top-N abstract-only, non-paywalled candidates. CRON-2 (`/api/cron/restage-with-bodies`), a few minutes later in the same daily cycle, re-runs the real `generateDailyBriefing` pipeline with those bodies merged (via `useExtractedBodies`), then appends the newly-eligible items into today's `signal_posts` and re-runs staging so the Notion editorial queue sees them this cycle. Product scope is PRD-70; this ADR records the engineering decision and the alternatives rejected.

The context is a four-round read-only diagnosis. Importance was already corrected (PRD-38), so genuine news scores 52–80, yet `getExclusionCause` returns `source_accessibility` first — before importance is consulted — and a measured first-gate attribution found it kills ~10 of 11 genuine clusters on a representative day because the bodies never reach the gate. The pipeline already holds each article URL but never fetches the body. Two hard constraints shaped the design: the eligibility gate is entangled with fresh ingestion (it consumes the in-memory `StoryCluster` + full `RankedStoryClusterResult`, neither persisted, so it is not re-entrant on persisted candidate rows), and each Vercel function is bounded at 60 seconds (the main run already flirts with 55–58 s).

Four alternatives were considered. (A-single) one extraction route doing fetch + a full real re-run needs ~67 s and an unconfirmed ≥120 s plan cap that cannot be verified or preview-tested — a production-timeout risk — so it was rejected. (B) reconstructing `RawItem`s from `pipeline_article_candidates`' lossy scalar projection so the gate could be re-run without re-ingesting was rejected as the most dangerous option: any mis-reconstructed field yields silently-wrong eligibility, the exact malignant failure class the diagnosis discipline targets. (Lazy next-run re-eval) letting the next day's main run consume the cached bodies was rejected because the ~24 h lag defeats the breaking-news target — fast-moving URLs roll off the feed before the next run. (C) splitting `generateDailyBriefing` into ingest/evaluate phases is the correct long-term re-entrant refactor but is large and not single-concern, so it is deferred.

The chosen option is the two-phase split (A-2-phase). It reuses the real gate semantics end-to-end (no reconstruction), provably fits the known 60 s cap per leg (fetch ~12 s in CRON-1; one ingestion+staging envelope ~55 s in CRON-2), and keeps `fetch-editorial-inputs` byte-for-byte unchanged because the body map is gated on an explicit `useExtractedBodies` flag only CRON-2 sets. Its failure modes are observable and benign: it degrades to a no-op pre-migration and on any error leaves today's slate exactly as the main run produced it. The one persistence deviation is a contained URL-keyed append-persist (`persistSignalPostCandidates` mode `"append"`): the existing rank-keyed snapshot guard is position-based and would drop the highest-importance newly-unblocked items on a re-run, so append mode inserts only `canonical_url`s absent for today, at post-max ranks capped at the `rank ≤ 20` CHECK, never overwriting or duplicating existing (including human-edited) rows.

The accepted consequences are that extraction touches only non-paywalled sources (paywalled full text stays correctly ungroundable), the daily run now costs a second feed fetch in CRON-2's isolated budget, and appended items take post-max ranks — queue availability, not importance ordering, which stays under human control. The decision should be revisited when the team is ready for option C (split `generateDailyBriefing` into ingest/evaluate phases), which would let the re-gate run on the exact current-cycle candidates without a second ingestion, or if daily volume ever pushes appended ranks against the 20-row cap.
